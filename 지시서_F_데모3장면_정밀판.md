# 코다리 지시서 — F항목: 데모 3장면 확장 (정밀판, 페이블 설계 유산)

> 우선순위: 결제 오픈 후 첫 콘텐츠 작업. 예상 소요 1~2시간.

## 목표
demo.html의 고래(영상) 1장면 → 고래 + 등대 + 돛단배 3장면. 등대·돛단배는 영상이 아니라 **선이 그려지는 SVG 모드**.

## 데이터 (이미 서버에 있음 — 새로 만들지 말 것)
- `/metric-scenes.json`의 lighthouse(87획)·ship(113획) lineart 배열: `{d:"M ...", t:""}` 형태의 SVG path
- 원화 이미지: `/metric-lighthouse.png`, `/metric-ship.png` (누끼 RGBA)

## 구현 (demo.html 안에 바닐라 JS)
1. 장면 전환 UI: 스테이지 아래 점 3개(● ○ ○). 탭하면 장면 교체. 교체 시 resetScene() 동작 보장.
2. 고래 장면: 기존 영상 방식 그대로 (수정 금지).
3. 등대·돛단배 장면 (SVG 모드):
   a. viewBox는 json의 vb 값 사용. stroke #1a1a1a, stroke-width는 json의 sw, fill none, stroke-linecap round.
   b. 【헌법】모든 획을 strokeDasharray/dashoffset으로 **순차** 드로잉. 획당 최소시간 클램프 금지.
      속도 공식: perStrokeMs[i] = strokeLen[i] / totalLen × (6000 − totalPauseMs). 획 사이 pause 20~60ms 랜덤.
   c. 【헌법】전 획 완주 후에만 → 원화 PNG 크로스페이드(blur 3→0, scale 1.015→1, 0.8s) → 단어 가족 팝+TTS.
4. 단어 가족 (그림 진실의 법칙 — PNG를 열어 실제 보이는 것만 확정할 것):
   - LIGHTHOUSE 등대 / LIGHT 빛 / SEA 바다 / ROCK 바위 ← 그림에 없는 요소는 뺀다
   - SHIP 돛단배 / SAIL 돛 / WAVE 파도 / MAST 돛대 ← 동일
   - 좌표는 고래 방식(top/left %)으로 각 부위 근처에. TTS는 기존 speak() 재사용 (unlockSpeech 유지).
5. 케이던스: 장면당 그리기 6초 + 단어 팝. "지금은 맛보기 장면" 카피를 "맛보기 장면 3개"로 수정 (이 단어만 변경 허용).
