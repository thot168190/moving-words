## A. 【버그·최우선】 히어로 영상 뚝뚝 끊김 — 원인: autoplay와 마우스 스크럽이 서로 싸움

**진단(확정)**: src/App.tsx에 히어로 비디오가 `autoPlay loop`로 재생되는 동시에,
mousemove → `video.currentTime = current` 를 매 프레임(rAF) 실행하는 스크럽 useEffect가 살아 있다.
재생 헤드를 두 시스템이 동시에 움직여서 화면이 튄다.

**수정**: 마우스 스크럽 useEffect **통째로 삭제** (`onMove`, `tick`, `requestAnimationFrame(tick)`, heroVideoRef 관련 mousemove 리스너 전부).
비디오 태그는 그대로: `playsInline muted autoPlay loop preload="auto"`.
heroVideoRef가 다른 곳에서 안 쓰이면 ref도 제거.

## B. 【버그·최우선】 왼쪽 한글 카피가 화면 앞(왼쪽 끝)에 잘려 붙는 문제 — 원인: h1 nowrap 가로 오버플로

**진단(확정)**: 히어로 h1에 `whiteSpace: 'nowrap'` + `clamp(32px,5vw,72px)`.
제목이 길어 뷰포트보다 넓어지면 가로 스크롤이 생기고, 페이지가 옆으로 밀리며 왼쪽 카피가 화면 밖으로 나간다.

**수정**:
1. 히어로 h1의 `whiteSpace: 'nowrap'` 제거 → 대신 `wordBreak: 'keep-all'` 추가 (줄바꿈은 기존 <br/>가 담당)
2. 최상위 컨테이너(또는 body)에 `overflow-x-hidden` 보증: `<div className="... overflow-x-hidden">`
3. **검증**: 브라우저 폭 1280px과 800px에서 가로 스크롤바가 없어야 함

## I. 【버그·최우선】 모바일에서 데모 발음(TTS) 무음 — 원인: 사용자 제스처 밖에서 speak() 호출

**진단(확정)**: demo.html의 speak()는 영상 ended 이벤트 → setTimeout 안에서 실행된다.
iOS 사파리·카카오 인앱 브라우저는 **사용자 터치 핸들러 안에서 시작되지 않은 음성 재생을 차단**한다.
터치(클릭) 시점에 오디오를 한 번 "해금"해두면 이후 타이머 안의 speak()가 허용된다.

**수정 — demo.html <script>에 정확히 다음을 적용**:

1. 스크립트 상단에 해금 함수 추가:
```js
let speechUnlocked = false;
function unlockSpeech() {
  if (speechUnlocked || !window.speechSynthesis) return;
  const u = new SpeechSynthesisUtterance(' ');
  u.volume = 0;
  window.speechSynthesis.speak(u);   // 제스처 안에서 1회 재생 → 이후 타이머 speak 허용
  speechUnlocked = true;
}
```

2. 기존 `sceneStage.addEventListener('click', () => {` 의 **첫 줄**에 `unlockSpeech();` 추가.
   (resetScene 분기보다 먼저 — 어떤 분기로 가든 해금되도록)

3. 모바일은 voices 목록이 비동기 로드라 getBestVoice()가 undefined일 수 있음 — 현재 코드는
   `if (voice) u.voice = voice;` 라 기본 보이스로 폴백되므로 **speak() 로직은 수정 금지**. 대신
   `window.speechSynthesis.onvoiceschanged = () => {};` 를
   `window.speechSynthesis.getVoices();` 호출이 포함된 형태로 유지해 목록 로드를 트리거.

4. resetScene()의 `speechSynthesis.cancel()`은 유지 (연타 시 중복 발화 방지용, 해금과 무관).

**한계 고지(정직)**: 카카오 인앱(iOS WKWebView)은 위 해금으로도 TTS가 안 나오는 기기가 있다.
이 경우의 정공법은 **단어별 MP3 사전 생성**(토요일 백로그 항목)이며, 이번 라운드에서는 해금 패치까지만.
